## 2. 实体定义

### 2.1 DatabaseConnection (数据库连接)

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| ConnectionId | string(50) | PK | 连接ID，主键 |
| Name | string(100) | NOT NULL | 连接名称 |
| Type | int | NOT NULL | 数据库类型枚举 |
| Host | string(255) | NULLABLE | 主机地址 |
| Port | int | NULLABLE | 端口号 |
| Database | string(100) | NULLABLE | 数据库名 |
| Username | string(100) | NULLABLE | 用户名 |
| EncryptedPassword | string(500) | NULLABLE | AES加密密码 |
| MaxConnections | int | DEFAULT 10 | 最大连接数 |
| CommandTimeout | int | DEFAULT 30 | 命令超时(秒) |
| IsEnabled | bool | DEFAULT true | 是否启用 |
| CreatedAt | datetime | NOT NULL | 创建时间 |
| LastConnectedAt | datetime | NULLABLE | 最后连接时间 |
| UpdatedAt | datetime | NULLABLE | 更新时间 |

**索引:**
- PRIMARY KEY (ConnectionId)
- INDEX idx_name (Name)
- INDEX idx_type (Type)

**示例数据:**
```json
{
    "connectionId": "conn_001",
    "name": "生产数据库",
    "type": 1,
    "host": "192.168.1.100",
    "port": 5432,
    "database": "mydb",
    "username": "readonly_user",
    "maxConnections": 10,
    "commandTimeout": 30,
    "isEnabled": true,
    "createdAt": "2026-04-01T10:00:00Z"
}
```

## 6. Redis Key设计

### 6.1 Key命名规范

```
格式: dqdb:{category}:{subcategory}:{identifier}:{qualifier}

命名规则:
- 使用冒号(:)分隔层级
- 使用下划线(_)连接单词
- identifier唯一
- qualifier可选(用于版本、时间等)
```

### 6.2 Key定义

| Key模式 | 用途 | TTL | 说明 |
|---------|------|-----|------|
| `dqdb:meta:tables:{connectionId}` | 表信息缓存 | 5分钟 | Schema表信息 |
| `dqdb:meta:schemas:{connectionId}` | Schema缓存 | 5分钟 | Schema列表 |
| `dqdb:ai:sessions:{sessionId}` | AI会话 | 10分钟 | 会话数据 |
| `dqdb:ai:context:{sessionId}` | 会话上下文 | 10分钟 | 上下文窗口 |
| `dqdb:cache:query:{hash}` | 查询结果缓存 | 30分钟 | 重复查询优化 |
| `dqdb:rate:api:{userId}` | API限流 | 1分钟 | 限流计数 |
| `dqdb:rate:ai:{userId}` | AI限流 | 1分钟 | AI限流计数 |
| `dqdb:lock:conn:{connectionId}` | 连接锁 | 30秒 | 连接操作锁 |

### 6.3 缓存策略

| 缓存类型 | 策略 | 说明 |
|---------|------|------|
| Schema缓存 | LRU + TTL | 5分钟TTL |
| AI会话 | LRU + TTL | 10分钟TTL |
| 查询结果 | LRU + TTL | 30分钟TTL |
| 限流计数 | TTL | 1分钟TTL |
| 分布式锁 | TTL | 按需设置 |

## 8. 数据库迁移

### 8.1 初始迁移

```sql
-- Initial Migration

CREATE TABLE IF NOT EXISTS DatabaseConnections (
    ConnectionId TEXT PRIMARY KEY,
    Name TEXT NOT NULL,
    Type INTEGER NOT NULL,
    Host TEXT,
    Port INTEGER,
    Database TEXT,
    Username TEXT,
    EncryptedPassword TEXT,
    MaxConnections INTEGER DEFAULT 10,
    CommandTimeout INTEGER DEFAULT 30,
    IsEnabled INTEGER DEFAULT 1,
    CreatedAt TEXT NOT NULL,
    LastConnectedAt TEXT,
    UpdatedAt TEXT
);

CREATE TABLE IF NOT EXISTS AiSessions (
    SessionId TEXT PRIMARY KEY,
    ConnectionId TEXT NOT NULL,
    Title TEXT NOT NULL,
    Status INTEGER DEFAULT 0,
    MessageCount INTEGER DEFAULT 0,
    CreatedAt TEXT NOT NULL,
    LastActivityAt TEXT NOT NULL,
    ExpiresAt TEXT,
    FOREIGN KEY (ConnectionId) REFERENCES DatabaseConnections(ConnectionId)
);

CREATE TABLE IF NOT EXISTS AiMessages (
    MessageId TEXT PRIMARY KEY,
    SessionId TEXT NOT NULL,
    Role INTEGER NOT NULL,
    Content TEXT NOT NULL,
    Sql TEXT,
    Explanation TEXT,
    Confidence REAL,
    CreatedAt TEXT NOT NULL,
    FOREIGN KEY (SessionId) REFERENCES AiSessions(SessionId)
);

CREATE TABLE IF NOT EXISTS QueryHistory (
    HistoryId TEXT PRIMARY KEY,
    ConnectionId TEXT NOT NULL,
    Sql TEXT NOT NULL,
    NaturalLanguage TEXT,
    ExecutionTime INTEGER,
    RowCount INTEGER,
    Status INTEGER NOT NULL,
    ErrorMessage TEXT,
    CreatedAt TEXT NOT NULL,
    FOREIGN KEY (ConnectionId) REFERENCES DatabaseConnections(ConnectionId)
);

CREATE INDEX idx_connections_name ON DatabaseConnections(Name);
CREATE INDEX idx_connections_type ON DatabaseConnections(Type);
CREATE INDEX idx_sessions_connection ON AiSessions(ConnectionId);
CREATE INDEX idx_sessions_activity ON AiSessions(LastActivityAt);
CREATE INDEX idx_messages_session ON AiMessages(SessionId);
CREATE INDEX idx_history_connection ON QueryHistory(ConnectionId);
CREATE INDEX idx_history_created ON QueryHistory(CreatedAt);
```
