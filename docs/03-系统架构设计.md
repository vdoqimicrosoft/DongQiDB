# 系统架构设计文档

## 1. 架构概览

### 1.1 架构风格: Clean Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Presentation Layer (API)                        │
│  ┌─────────────────────────────────────────────────────────────────────┐  │
│  │   Controllers    │    Middleware     │     Swagger / OpenAPI        │  │
│  └─────────────────────────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────────────────────────┤
│                              Application Layer                               │
│  ┌───────────────┬───────────────┬───────────────┬─────────────────────┐  │
│  │  AI Services  │ Query Services │ DDL Services  │  Export Services   │  │
│  └───────────────┴───────────────┴───────────────┴─────────────────────┘  │
├─────────────────────────────────────────────────────────────────────────────┤
│                              Domain Layer                                    │
│  ┌───────────────┬───────────────┬───────────────┬─────────────────────┐  │
│  │   Entities    │  Value Objects│   Interfaces  │   Domain Events    │  │
│  └───────────────┴───────────────┴───────────────┴─────────────────────┘  │
├─────────────────────────────────────────────────────────────────────────────┤
│                          Infrastructure Layer                                │
│  ┌───────────────┬───────────────┬───────────────┬─────────────────────┐  │
│  │ DB Providers  │  AI Clients   │    Redis     │  File Exporters    │  │
│  │ (EF Core)     │ (Anthropic)    │   Cache      │  (CSV/Excel)       │  │
│  └───────────────┴───────────────┴───────────────┴─────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 项目结构

```
DongQiDB.Client/
├── src/
│   ├── DongQiDB.Client.API/              # API层 - Controllers, Middleware
│   ├── DongQiDB.Client.Application/      # 应用层 - Services, Use Cases

│   └── DongQiDB.Client.Common/           # 公共层 - DTOs, Enums, Utils
```

## 5. 缓存设计

### 5.1 缓存策略

```
┌─────────────────────────────────────────────────────────────┐
│                        缓存层级                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   L1: 内存缓存 (LRU + TTL)                                  │
│       - Schema信息: TTL 5分钟                               │
│       - AI会话: TTL 10分钟                                  │
│       - 查询结果: TTL 30分钟                                 │
│                                                             │
│   L2: Redis缓存 (分布式)                                    │
│       - 跨实例共享                                          │
│       - 长时间缓存                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 Redis Key规范

```
dqdb:{category}:{subcategory}:{identifier}

示例:
dqdb:meta:tables:conn_123      # 连接123的表信息
dqdb:meta:schemas:conn_123     # 连接123的Schema
dqdb:ai:sessions:user_456      # 用户456的AI会话
dqdb:cache:query:hash_abc      # 查询结果缓存
```

## 9. 数据库路由策略

### 9.1 加权轮询

```csharp
public class DatabaseRouter
{
    private readonly WeightedRoundRobin<string> _readRouter;
    private readonly WeightedRoundRobin<string> _writeRouter;
    
    // 根据SQL类型选择连接
    public string SelectConnection(string sql)
    {
        if (IsReadQuery(sql))
            return _readRouter.Select();
        return _writeRouter.Select();
    }
}
```
